### 📋 git 코드 컨벤션
* feat: 새로운 기능 추가
* fix: 버그 수정
* docs: 문서 관련 변경 (예: README 수정)
* style: 코드 스타일 변경 (공백, 세미콜론 등 비즈니스 로직 변경 없음)
* refactor: 리팩토링 (기능 변경 없이 코드 구조 개선)
* test: 테스트 추가 또는 수정
* chore: 빌드 또는 기타 설정 변경 (코드 수정 없음)
* perf: 성능 개선 관련 변경
* ci: CI 설정 변경 (예: GitHub Actions, Jenkins 등)
* revert: 이전 커밋 되돌리기
